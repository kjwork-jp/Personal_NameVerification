# 91_uat_execution_plan_20260526.md

## Purpose

This document defines the UAT execution plan for NameVerification after the implementation / documentation / maintenance backlog was completed and Quality Gates passed.

This document starts the UAT preparation gate. It does not publish a release.

## Baseline

| Item | Value |
|---|---|
| Stable baseline | v0.2.0 |
| Development line | v0.3.0+ |

## UAT goals

| Goal | Description |
|---|---|
| Role validation | Confirm viewer / editor / admin behavior differences. |
| Data operation validation | Confirm search, CRUD, relation, delete/restore, export, backup, import preview, and audit workflows. |
| Safety validation | Confirm destructive operations require admin and operator awareness. |
| Documentation validation | Confirm manuals and Help / Settings text match current UI behavior. |
| Evidence validation | Confirm UAT evidence can be recorded without publishing a release. |

## Non-goals

- Do not create a GitHub Release.
- Do not tag a new version.
- Do not run release packaging as a publication step.
- Do not commit generated DB / CSV / export / backup files.
- Do not use production or personal sensitive data.

## Exit criteria

UAT can be considered complete only when all of the following are true:

1. All required checklist items are marked PASS or explicitly accepted as known limitations.
2. No unresolved P1/P2 blocker remains.
3. Any UAT finding is classified as blocker / fix before release / backlog / accepted limitation.
4. Release remains deferred until UAT findings are reviewed.
5. No generated UAT data is committed.

## Finding classification

| Classification | Meaning | Next action |
|---|---|---|
| blocker | Prevents safe UAT completion | Fix before continuing UAT |
| fix before release | UAT can continue, but release cannot start | Create implementation backlog |
| backlog | Not blocking release decision | Track after UAT |
| accepted limitation | Known and documented behavior | Record in UAT evidence |
| documentation fix | UI works, docs unclear or stale | Update manual/help docs |

## Evidence policy

- Keep raw generated DB/CSV/export/backup files out of git.
- Evidence notes may be committed as markdown if they contain no sensitive data.
- Screenshots should be avoided unless necessary; if used, confirm no sensitive values are visible.
- UAT outputs under `tmp/` are local only.
