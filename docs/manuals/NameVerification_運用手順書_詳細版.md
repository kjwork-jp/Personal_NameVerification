# NameVerification 運用手順書（詳細版）

## 1. 目的
運用担当が Day0/Day1 で迷わず実施できる詳細手順を示す。

## 2. 導入
1. GitHub から ZIP を取得して展開。
2. 作業ディレクトリを固定。
3. 依存導入後にアプリ起動。
4. タブ（検索/名前管理/タイトル/サブタイトル管理/リンク管理/ゴミ箱/監査ログ/運用操作）を確認。

## 3. 日次業務フロー
1. 検索
2. 名前管理
3. タイトル/サブタイトル管理
4. リンク管理
5. ゴミ箱/監査ログ確認
6. 運用操作（export/backup）

## 4. タイトル作成と名前紐づけ
1. タイトル作成前に名前一覧から複数選択。
2. タイトル作成を実行。
3. 紐づき名前表示で結果確認。

## 5. ID 運用方針
- ID は DB 自動採番。
- 新規作成時に手入力不要。
- 更新/削除対象は一覧選択で指定。

## 6. name_title_links 運用
- direct many-to-many を保持。
- export/import に含まれるため運用バックアップの整合性を維持可能。

## 7. 運用操作タブ
- 参照ボタンでパス指定。
- 実行前確認が必要な destructive 操作（restore/import）は admin のみ。
- ログビューアで操作履歴を確認。

## 8. RBAC
- viewer: 閲覧のみ
- editor: 作成/更新/link 可、destructive 不可
- admin: destructive 含む全操作可

## 9. 一次切り分け
- 起動不可 / DB ロック / import 条件違反 / ログ未表示を順に確認。
