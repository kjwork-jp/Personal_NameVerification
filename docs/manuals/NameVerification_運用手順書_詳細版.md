# NameVerification 運用手順書（詳細版）

## 1. 目的

運用担当が Day0 / Day1 で迷わず実施できる詳細手順を示します。
この手順書は通常運用向けです。release作業、UAT、CI/CD詳細は扱いません。

## 2. Day0: 初期準備

1. 配布物またはリポジトリを所定フォルダに配置する。
2. Python実行の場合は依存関係を導入する。
3. EXE配布の場合はportable release layoutを崩さず配置する。
4. DB / logs / backups / exports の保存先を確認する。
5. 必要に応じてOS ACL / BitLocker / EFS / 共有権限を確認する。
6. アプリを起動する。
7. 初回adminが存在しない場合は、初回adminを作成する。
8. adminでログインできることを確認する。

## 3. Day1: 通常起動

1. アプリを起動する。
2. operator_id と password でログインする。
3. status表示でログイン中ユーザーとroleを確認する。
4. 「ヘルプ/設定」でDB保存先とlog保存先を確認する。
5. 「検索」で既存データの表示を確認する。

## 4. 日次業務フロー

1. 「検索」で対象データを確認する。
2. 「名前を管理」で名前を作成・更新する。
3. 「タイトル/サブタイトル管理」でタイトル・サブタイトルを作成・更新する。
4. 必要な場合のみ「関連付け」で名前とサブタイトルを関連付ける。
5. adminのみ「削除データ」で復元/完全削除を行う。
6. 「操作履歴」で変更履歴を確認する。
7. 「データ入出力」でbackup/exportを実行する。

## 5. 一覧起点操作

原則として、更新・削除対象は一覧から選択します。

| 画面 | 操作方針 |
|---|---|
| 名前を管理 | 名前一覧を選択し、フォームで確認・更新する |
| タイトル/サブタイトル管理 | タイトル一覧・サブタイトル一覧を選択し、フォームで確認・更新する |
| 削除データ | 削除済みデータ一覧から復元/完全削除対象を選択する |
| 操作履歴 | 履歴一覧・検索条件・差分表示を確認する |

IDはDB自動採番です。新規作成時に手入力しません。

## 6. 名前作成/更新

1. 「名前を管理」を開く。
2. 新規作成時は入力欄へ名前と備考を入力する。
3. 更新時は一覧から対象を選択する。
4. フォームに反映された内容を確認する。
5. 作成または更新を実行する。
6. message表示と一覧を確認する。

## 7. タイトル/サブタイトル作成/更新

1. 「タイトル/サブタイトル管理」を開く。
2. タイトル一覧を確認する。
3. タイトル新規作成時はタイトル名・備考・必要な関連名を入力する。
4. タイトル更新時はタイトル一覧から対象を選択する。
5. サブタイトルを扱う場合は、まず対象タイトルを選択する。
6. サブタイトル一覧を確認する。
7. サブタイトル名、管理番号、表示順、備考を入力または更新する。
8. 作成または更新を実行する。

## 8. 関連付け

1. 「関連付け」を開く。
2. 名前とサブタイトルを選択する。
3. 関連登録を実行する。
4. adminの場合のみ、必要に応じて関連解除を実行する。
5. 検索画面または関連明細で結果を確認する。

## 9. 削除/復元/完全削除

削除系は原則admin操作です。

1. 対象画面でゴミ箱投入を行う。
2. 「削除データ」を開く。
3. 復元または完全削除対象を選択する。
4. 確認ダイアログを確認する。
5. 実行後、一覧・検索・操作履歴を確認する。

完全削除は取り消しできません。実行前にbackup取得を推奨します。

## 10. データ入出力

### 10.1 Export

1. 「データ入出力」を開く。
2. CSV / JSON / SQL dump の出力先を確認する。
3. 実行する。
4. 出力ファイルが保存先に作成されたことを確認する。

注意:

- SQL dump は full DB dump として扱う。
- 共有用JSON出力は認証・管理・設定系テーブルを除外する用途。

### 10.2 Backup

1. 「データ入出力」を開く。
2. backup出力先を確認する。
3. backupを実行する。
4. `50_backups` 配下または指定先にDB backupが作成されたことを確認する。

### 10.3 Restore

1. adminでログインする。
2. restore対象backupを確認する。
3. 実行前に現在DBの退避backupが作成されることを理解する。
4. restoreを実行する。
5. 起動・検索・件数・操作履歴を確認する。

### 10.4 Import

1. adminでログインする。
2. import対象ファイルを確認する。
3. 実行前にbackupを取得する。
4. importを実行する。
5. 件数・検索・操作履歴を確認する。

## 11. 操作履歴/監査ログ

| 画面 | 確認内容 |
|---|---|
| 操作履歴 | DB内 `change_logs`。通常データ変更の確認 |
| ユーザー監査ログ | login、user作成、role変更、有効/無効変更などの確認 |
| Operations log | データ入出力タブの実行JSONLログ確認 |

## 12. RBAC運用

| role | 運用上の割当例 |
|---|---|
| viewer | 参照担当、確認担当 |
| editor | 日常登録・更新・export/backup担当 |
| admin | 削除、復元、import、restore、ユーザー管理担当 |

adminは必要最小限にします。

## 13. サンプルDB作成

UAT / デモ用の小規模DB:

```powershell
python .\scripts\generate_sample_data.py --preset demo --format sqlite --output tmp\demo.db
```

CSV確認用の小規模データ:

```powershell
python .\scripts\generate_sample_data.py --preset demo --format csv --output tmp\demo_csv
```

## 14. ファイル保護チェック

1. 「ヘルプ/設定」を開く。
2. DB、log、backup、export、SQL dumpの保存先を確認する。
3. Windowsの場合は `Get-Acl` または `icacls` で権限を確認する。
4. Users / Authenticated Users / Everyone に不要な読取権限がないか確認する。
5. 必要に応じてBitLocker / EFS / 共有権限を併用する。

## 15. 一次切り分け

| 事象 | 確認順 |
|---|---|
| 起動不可 | 配置、Python/EXE、依存、DBパス、権限 |
| ログイン不可 | operator_id、password、user有効状態、role |
| ボタンが押せない | role、対象選択、削除済み状態、admin専用操作 |
| データが見えない | 検索条件、削除済み状態、DB保存先 |
| exportされない | 出力先、権限、role、エラーメッセージ |
| restore/import失敗 | admin権限、backup、DBロック、入力ファイル形式 |
| ログが見えない | filter、ページング、保存先、JSONLファイル存在 |

## 16. 禁止・注意

- release準備とUATは、全残件完了後まで着手しない。
- 実データをリポジトリに置かない。
- DB / backup / export / SQL dump を不用意に共有しない。
- 完全削除前はbackupを取得する。
