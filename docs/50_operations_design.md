# 50_operations_design.md

## 1. 目的
運用開始後に必要な日次・週次・障害時運用を定義する。

## 2. 日次運用
- 起動確認
- 代表検索確認
- 変更ログ確認
- 必要時の CSV / JSON / SQL dump 出力（editor/admin）
- 終業前バックアップ作成（editor/admin）

## 3. 週次運用
- バックアップ復元リハーサル（復元機能は後続実装）
- 運用レポート確認
- change_logs spot review
- export / backup 出力ファイル妥当性サンプリング
- 未処理課題確認

## 4. 月次運用
- バックアップ世代確認
- 権限設定棚卸し
- 障害 / 問い合わせ傾向レビュー

## 5. 運用ロール
- 運用担当
- 管理者
- 開発担当
