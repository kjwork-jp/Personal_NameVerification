# 10_requirements_definition.md

## 1. システム目的
利用者が、名前を起点として関連タイトル・サブタイトルの照合、登録、更新、削除、復元、移行、監査確認を一貫して行える Windows デスクトップアプリを提供する。

## 2. 利用前提
- Windows 11 ローカル端末
- 単一端末での業務利用を主前提
- オフライン運用可能
- SQLite ローカルファイルで運用
- GUI は PySide6

## 3. 業務上の課題
- 名前の表記ゆれにより重複判断が難しい
- 関連タイトル/サブタイトルの紐づき確認が煩雑
- 誤登録時の修正・追跡がしにくい
- Excel 等の散在管理では整合性が崩れやすい

## 4. 利用者ロール
- viewer: read-only 操作
- editor: read-only + write 操作
- admin: read-only + write + destructive 操作

## 5. 機能要件
### 5.1 検索
- 名前検索を主導線とする
- 完全一致 / 部分一致を切替可能
- タイトル条件で絞込可能
- 紐づき有無で絞込可能
- ソート条件を指定可能
- 検索結果から詳細確認できる

### 5.2 名前管理
- 名前の作成
- 名前の更新
- 論理削除
- 復元
- 完全削除
- 正規化後一意制御

### 5.3 タイトル管理
- タイトルの作成
- 更新
- 論理削除
- 復元
- 完全削除

### 5.4 サブタイトル管理
- サブタイトルの作成
- 更新
- 論理削除
- 復元
- 完全削除
- title 配下の順序制御

### 5.5 紐づき管理
- 名前とサブタイトルの関連付け
- 関連解除
- 関連一覧表示

### 5.6 アイコン管理
- ファイル選択による添付
- 不正形式の拒否
- 過大サイズの拒否
- パス保持またはバイナリ保持の設計ルールに従う

### 5.7 データ移行
- CSV Export / Import
- JSON Export / Import
- SQL dump Export
- バックアップ作成
- バックアップ復旧

### 5.8 監査・運用
- 操作者ID を含む変更履歴の記録
- change_logs の参照と絞込
- 運用レポート出力
- UAT 実行用補助導線
- 障害対応手順へのリンク

### 5.9 RBAC（現行最小方針）
- valid role（viewer/editor/admin）は read-only 操作を実行できる。
- write 操作（作成/更新/リンク操作）は editor/admin のみ実行できる。
- destructive 操作（論理削除/復元/完全削除）は admin のみ実行できる。
- ゴミ箱/削除済み一覧および監査ログは「閲覧」と「操作」を分離し、閲覧は read-only、復元/完全削除等は destructive として扱う。

## 6. 非機能要件
### 6.1 性能
- 通常操作で待ち時間が目立たない
- 検索、更新、削除、復元が実務上許容される速度で完了する

### 6.2 可用性
- バックアップ手順が存在する
- 復旧手順が定義される
- 破壊的操作前に確認導線がある

### 6.3 保守性
- UI 層 / サービス層 / DB 層を分離する
- ドキュメントと実装の整合を維持する
- 主起動点は 1 つだけにする

### 6.4 監査性
- 誰が何を変更したか追跡可能
- 削除、復元、完全削除は明確に区別される
- 運用開始前にレビュー証跡が残る

## 7. 制約
- 既定 UI は PySide6 のみ
- Tkinter を併存しない
- 外部クラウド依存を前提にしない
