# 00_repository_master_spec.md

## 1. 文書の目的
本書は、NameVerification v3 リポジトリの最上位設計書です。  
リポジトリ構成、目的、対象範囲、品質基準、文書体系、開発原則を定義します。

## 2. リポジトリの目的
本リポジトリは、Windows ローカル環境で動作する**名前照合・登録アプリ**を、PySide6 を正式 UI としてゼロから再構築するための唯一の正本です。

## 3. 成果物
本リポジトリが最終的に持つべき成果物は以下です。
- 実行可能なデスクトップアプリ
- 単体 / 結合 / UAT テスト
- バックアップ / 復旧 / 監査 / 運用文書
- リリース可能な配布物
- 開発・保守・障害対応が継続できる文書群

## 4. 想定ユーザー
- 業務担当者
- 運用担当者
- 開発担当者
- レビュー担当者
- 将来の引継ぎ担当者

## 5. 基本方針
- 仕様と実装の乖離を最小化する
- 文書先行で設計を固定し、その後に実装する
- main は常にレビュー可能な状態を保つ
- 設計変更は痕跡を残す
- 各工程末尾に**徹底的な最終見直しレビュー**を行う

## 6. 採用技術方針
- UI: PySide6
- Language: Python 3.12 系
- Database: SQLite
- Package: PyInstaller
- Test: pytest 系を基本とする
- Lint/Format/Type: ruff, black, mypy

## 7. 非採用方針
- Tkinter 併存
- Web 化
- クラウド前提
- 複数 DB 対応
- 未承認の外部 API 依存

## 8. 管理原則
- 1 PR 1目的
- 文書優先順位は `docs/02_document_priority_order.md` に従う
- 役割分担は `docs/01_role_responsibility_matrix.md` に従う
- 不明点は `docs/97_open_issues_and_constraints.md` に記録する

## 9. リポジトリ標準ディレクトリ
### 現時点
- `docs/`: 設計・品質・運用文書
- `prompts/`: Codex 向け指示文書
- ルート: ガバナンス・メタ情報

### 実装開始後の想定
- `app/`: アプリ本体
- `tests/`: テスト
- `scripts/`: 運用補助スクリプト
- `db/`: スキーマ・マイグレーション
- `assets/`: アイコン等

## 10. 成功条件
- PySide6 正式版が単独で成立する
- ドキュメント体系が破綻していない
- AI と人間の役割分担が明確
- リリースから運用開始までの判断材料が揃っている

## 11. 変更管理
本書を変更する場合、同時に以下の影響確認が必要です。
- 要件
- 基本設計
- 技術詳細設計
- 開発工程
- テスト戦略
- 運用設計
