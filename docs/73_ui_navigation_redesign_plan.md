# 73_ui_navigation_redesign_plan.md

## 1. 目的

NameVerification v3 のUIを、単純な横並びタブの集合から、業務導線に沿ったナビゲーション構造へ再設計する。

UAT中に `ユーザー管理` タブで以下の問題が顕在化した。

- 1画面に作成、一覧、更新系操作を詰め込みすぎている。
- 下部操作欄の目的が分かりづらい。
- ラベルと入力欄が画面サイズ・Qtテーマにより潰れる。
- 操作方法がタブごとに明示されていない。
- 主要タブが横並びで増えすぎ、業務フローが見えにくい。

このため、v0.2.0 UATの継続前に、UI構造の見直しを正式な改善工程として扱う。

---

## 2. 基本方針

| 方針 | 内容 |
|---|---|
| 1画面1責務 | 一覧、登録、編集、操作、履歴を1画面に混在させない |
| サブタブ分離 | 1オブジェクト内に複数業務がある場合、タブ内サブタブで分離する |
| 操作ガイド標準化 | 各タブ先頭に、何をどの順で操作するかを明示する |
| 一覧起点 | 編集・削除・復元などは原則一覧選択を起点にする |
| 状態表示統一 | 成功/警告/エラーを共通status componentで表示する |
| 危険操作隔離 | import / restore / hard delete / disable admin 等は通常操作から分離する |
| UAT容易性 | 各操作にUAT IDを割り当てやすい画面構造にする |

---

## 3. 現行トップレベルタブの問題

| 現行タブ | 問題 |
|---|---|
| 検索 | 検索条件、結果、詳細確認の導線整理が不足 |
| 名前を管理 | 一覧/登録/編集/削除の責務が混在しやすい |
| タイトルを管理 | 登録、編集、名前関連付けの導線が混在しやすい |
| サブタイトルを管理 | タイトル選択、サブタイトル一覧、登録/更新の階層が見えづらい |
| 関連付け | 対象選択、関連作成、解除、確認が同一画面に混在しやすい |
| 削除データ | 名前/タイトル/サブタイトル/linkの復元/完全削除が混在 |
| 操作履歴 | 検索条件、一覧、差分詳細の分離が不足 |
| ユーザー管理 | 作成/一覧/更新系操作を1画面に詰め込みすぎ。サブタブ化に着手済み |
| ユーザー監査ログ | filter、一覧、詳細差分の導線を整理すべき |
| データ入出力 | export/backup/restore/import/log が同一画面に混在。危険操作の分離が必要 |
| ヘルプ/設定 | ヘルプ、診断、環境変数、設定が混在しやすい |

## 4. 推奨UI構造

### 4.1 トップレベルカテゴリ

| カテゴリ | 画面 |
|---|---|
| 検索・参照 | 検索、詳細確認 |
| マスター管理 | 名前、タイトル、サブタイトル、関連付け |
| 運用管理 | データ入出力、バックアップ、復元、import、操作履歴 |
| セキュリティ管理 | ユーザー管理、ユーザー監査ログ |
| ヘルプ・診断 | ヘルプ、設定、パス診断、保護警告 |

トップレベルをすぐに全面変更すると影響が大きいため、当面は現行トップタブを維持しつつ、各タブ内をサブタブ化する。

### 4.2 各オブジェクトタブの標準サブタブ

| サブタブ | 用途 |
|---|---|
| ガイド | 対象機能の目的、操作順、注意点 |
| 一覧 | 登録済みデータの確認、対象選択 |
| 登録 | 新規登録 |
| 編集 | 選択済み対象の更新 |
| 操作 | 削除、復元、権限変更、関連解除などの状態変更 |
| 履歴/ログ | 対象操作の履歴確認 |

全タブに全サブタブを持たせる必要はない。対象オブジェクトの責務に応じて選択する。

## 7. 実装順序

| 順序 | 作業 | 目的 |
|---:|---|---|
| 1 | ユーザー管理サブタブ化のUAT完了 | 現在のblocker解消 |
| 2 | docs/72へUI blockerと修正履歴を反映 | UAT証跡を残す |
| 3 | データ入出力タブをExport/Backup/Restore/Import/Logに分割 | 危険操作を隔離 |
| 4 | 削除データタブを対象別に分割 | 復元/完全削除の誤操作を減らす |
| 5 | 名前/タイトル/サブタイトル管理を一覧起点へ再構成 | 日常CRUDの操作性改善 |
| 6 | 関連付けタブを対象選択/作成/解除/確認に分割 | 関連操作の迷いを減らす |
| 7 | 操作履歴/ユーザー監査ログの検索・一覧・詳細構造を統一 | ログ確認体験改善 |
| 8 | ヘルプ/設定を診断画面へ拡張 | DB/backup/export/log保護警告へ接続 |

## 8. UAT方針

UI再設計後は、機能UATだけでなく以下を確認する。

| ID | 観点 | 確認内容 |
|---|---|---|
| UI-UAT-001 | 視認性 | ラベル・入力欄・ボタンが隠れない |
| UI-UAT-002 | 導線 | 何を先に操作すべきか分かる |
| UI-UAT-003 | 一覧起点 | 一覧選択が編集/操作に反映される |
| UI-UAT-004 | 権限制御 | viewer/editor/adminで表示・操作が破綻しない |
| UI-UAT-005 | 危険操作隔離 | destructive操作が通常操作と分離されている |
| UI-UAT-006 | 画面サイズ耐性 | 1180x760相当で主要操作が見切れない |

## 9. 次工程

1. `ユーザー管理` サブタブ化後の品質ゲートとGUI確認を完了する。
2. `docs/72_v0_2_0_auth_integrated_uat_execution_record.md` にUI blocker修正と再開判断を追記する。
3. 次のUI改善対象として `データ入出力` と `削除データ` を優先する。
4. その後、通常CRUD系の `名前を管理` / `タイトルを管理` / `サブタイトルを管理` / `関連付け` へ横展開する。
