# 14_ui_ux_design.md

## 1. 目的
UI/UX 設計の基準を定義し、実装者ごとの差異を抑える。

## 2. UX 原則
- 最初にやりたい操作は検索である
- 主導線に余計な情報を置かない
- 詳細は二段目に出す
- destructive 操作は誤操作しづらくする
- エラーは利用者が次に何をすればよいか分かる文面にする

## 3. 画面原則
- タブ数は固定し、責務を混ぜない
- 一覧は最小列
- 詳細は別ペイン
- ステータスメッセージは下部固定
- キーボードショートカットを設ける

## 4. 表示原則
### 検索結果
- 主表示: 名前 / タイトル / コード / サブタイトル / 紐づき状態
- 補助表示: ID / 作成日時 / 更新日時 / operator_id

### フォーム
- 入力が必要な項目だけ表示
- システム値は直接編集させない
- 必須と任意を明確に分ける

## 5. ダイアログ原則
- 確認ダイアログ: 削除 / 復元 / 完全削除 / Import / Restore
- エラーダイアログ: 利用者が修正できる内容を優先
- 情報ダイアログ: 最小限

## 6. 文言原則
- ボタン文言は動詞ベース
- 「実行」「追加」「更新」「削除」「復元」を統一
- PoC、暫定、仮などの文言を残さない
