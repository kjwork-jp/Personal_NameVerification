## Included changes

- DB write transaction persistence fix.
- PySide6 UI redesign for Search, Name, Title, Subtitle, Relation, Trash, Audit, Operations, and Help/Settings tabs.
- Local login with role context and operator tracking.
- Automatic tab reload on tab switch.
- Cross-entity search across names, titles, subtitles, and subtitle codes.
- Title/subtitle related-count split.
- Name, title, subtitle, and relation management UI simplification.
- Trash tab consolidation for restore and hard delete.
- Human-readable audit log detail and diff rendering.
- Automatic DB change log JSONL export with rotation and best-effort failure handling.
- Operations tab compact two-column layout and destructive operation emphasis.
- README and in-app help refreshed for current operation model.
- Windows build and smoke test scripts validated locally.

## Distribution policy

- Do not commit `dist/NameVerification.exe` to the repository.
- Build the EXE locally from the fixed release candidate commit or branch.
- Keep the EXE, validation log, and release note together in the release package.
- Do not include personal SQLite databases in the release package.
- If sample data is required, regenerate it with `scripts/generate_sample_data.py` rather than committing generated data.
