# 20_icon_file_handling_spec.md

## 1. 目的
アイコンファイルの取扱いを統一する。

## 2. 方式
初期実装ではローカルファイルパスを主とし、必要に応じてバイナリ保存へ拡張する。

## 3. 許可形式
- png
- jpg
- jpeg
- webp

## 4. バリデーション
- 拡張子確認
- 実ファイル存在確認
- サイズ上限確認
- 読み取り可能確認

## 5. UI 方針
- 手入力ではなくファイル選択を原則とする
- 無効ファイル選択時は理由を表示する

## 6. 将来拡張
- DB BLOB 保存
- キャッシュ生成
- サムネイル表示
