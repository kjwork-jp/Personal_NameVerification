# 30_development_process_map.md

## 1. 目的
開発を工程単位で分割し、着手順序と責任を明確にする。

## 2. 工程一覧
1. リポジトリ bootstrap
2. 要件定義
3. 基本設計
4. 技術詳細設計
5. 実装準備
6. コア実装
7. UI 実装
8. 単体テスト
9. 初期結合テスト
10. 文書整合
11. UAT
12. 運用前総合レビュー
13. リリース準備
14. 運用開始判定

## 3. 担当分担
- 1〜4, 10〜14: ChatGPT 主担当
- 5〜9: Codex 主担当
- 全工程承認: Human Owner

## 4. 各工程末尾の必須作業
- セルフレビュー
- ドキュメント整合確認
- リスク記録
- **徹底的な最終見直しレビュー**

## 5. 工程ゲート
- Gate A: 要件承認
- Gate B: 設計承認
- Gate C: 実装承認
- Gate D: テスト承認
- Gate E: 運用開始承認
