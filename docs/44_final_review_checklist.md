# 44_final_review_checklist.md

## 目的
各工程末尾およびリリース直前に行う**徹底的な最終見直しレビュー**の基準。

## 1. 設計レビュー
- [ ] 文書間矛盾がない
- [ ] 非対象が明確
- [ ] 将来の削除計画がある

## 2. 実装レビュー
- [ ] PoC / 暫定命名が残っていない
- [ ] Tkinter 前提が残っていない
- [ ] 権限制御漏れがない
- [ ] destructive 操作が安全

## 3. テストレビュー
- [ ] 主要シナリオが通る
- [ ] 異常系が確認されている
- [ ] バックアップ / 復旧が確認済み

## 4. 運用レビュー
- [ ] Runbook が現実的
- [ ] ログ保存先が妥当
- [ ] 問い合わせ先が定義済み

## 5. リリースレビュー
- [ ] README / CHANGELOG / release 手順が一致
- [ ] ハッシュ確認手順あり
- [ ] マージ可否が説明できる
