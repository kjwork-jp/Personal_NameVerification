# 51_backup_restore_policy.md

## 1. バックアップ方針
- 日次バックアップ
- 世代管理
- 実ファイル存在確認
- 復元可能性を定期確認

## 2. バックアップ対象
- SQLite DB
- 監査ログ
- 設定ファイル
- 必要に応じて assets

## 3. 復旧方針
- 破壊的復旧前に現状退避
- 復旧後に整合性確認
- 復旧記録を残す

## 4. 禁止
- 未確認バックアップを本番復旧に使う


## 5. 実装ステータス（現時点）
- backup create（DB ファイルコピー）は実装済み。
- restore は未実装（後続PRで対応）。
- backup 実行権限は editor/admin、viewer は不可。
