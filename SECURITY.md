# SECURITY.md

## セキュリティ方針
NameVerification v3 はローカルデスクトップアプリですが、以下をセキュリティ対象とします。

- 個人名データ
- 監査ログ
- バックアップファイル
- 配布 exe
- 設計文書内の運用情報

## 報告対象
以下に該当する場合はセキュリティインシデントとして扱います。

- 権限を超えたデータ操作
- 監査ログ改ざん
- バックアップ漏えい
- 認可の抜け漏れ
- 任意ファイル読込 / 書込
- パス traversal
- 外部入力由来の破壊的操作

## 報告手順
1. 事象の再現条件を整理
2. 影響範囲を記録
3. 再現手順・証跡・ログを保全
4. `docs/55_incident_response_runbook.md` に従う
5. オーナーへ報告する

## 公開方針
- 脆弱性は修正前に公開しない
- issue / PR に詳細な exploit 手順を書かない
- 修正後に必要最低限の情報だけ残す

## 優先度
- Critical: 即時停止・修正
- High: 当日中に封じ込め
- Medium: 次回リリースまでに修正
- Low: 監視継続のうえ計画対応
